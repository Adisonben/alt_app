สรุปภาพรวม
--------------------------------------------------
- pysgfplib.py      = แกนหลัก (ขาดไม่ได้)
- sgfdxdevicename   = ระบุรุ่นเครื่อง
- sgfdxerrorcode    = จัดการ error
- sgfdxsecuritylevel= ปรับความเข้มการ match
- sgfplibtest.py    = ตัวอย่างอ้างอิง
- autoon.py         = สแกนอัตโนมัติ

ชุดไฟล์นี้เพียงพอสำหรับสร้างระบบลายนิ้วมือจริงระดับ Production



ไฟล์: pysgfplib.py
--------------------------------------------------
ไฟล์นี้คือ "หัวใจหลัก" ของการใช้ SecuGen SDK กับ Python

หน้าที่:
- เป็น Python Wrapper ที่ใช้ ctypes เรียก C Library ของ SecuGen
- เชื่อม Python → libpysgfplib.so → libsgfpamx.so

บทบาทหลัก:
- สร้างและทำลาย SGFPM Object
- Init / Open / Close เครื่องสแกน
- เปิด–ปิดไฟ LED
- สแกนลายนิ้วมือ (GetImage)
- ตรวจสอบคุณภาพภาพ
- สร้าง Fingerprint Template (SG400)
- เปรียบเทียบลายนิ้วมือ (Match)
- คำนวณ Matching Score

ใช้ตอนไหน:
- ทุกครั้งที่เขียนโปรแกรม Python ใช้งานเครื่องสแกน
- เป็นไฟล์ที่ต้อง import เสมอ

การประยุกต์ใช้งาน:
- ระบบ Kiosk
- ระบบ Login ด้วยลายนิ้วมือ
- Attendance / Check-in
- เชื่อมกับ Kivy, Flask, FastAPI


ไฟล์: sgfdxdevicename.py
--------------------------------------------------
ไฟล์นี้เก็บค่าคงที่ (Enum) สำหรับ "รุ่นของเครื่องสแกนลายนิ้วมือ"

หน้าที่:
- ระบุชนิดอุปกรณ์ให้ SDK รู้ว่าใช้งานรุ่นไหน

ค่าที่สำคัญ:
- SG_DEV_FDU03  = Hamster Plus
- SG_DEV_FDU04  = Hamster IV
- SG_DEV_FDU05  = Hamster PRO 20
- SG_DEV_AUTO   = ให้ SDK ตรวจจับอัตโนมัติ

ใช้ตอนไหน:
- ตอนเรียก Init() เพื่อเริ่มต้นอุปกรณ์

การประยุกต์ใช้งาน:
- ถ้ามีหลายรุ่น → ระบุรุ่นให้ตรง
- ถ้าเสียบเครื่องเดียว → ใช้ AUTO สะดวกที่สุด


ไฟล์: sgfdxerrorcode.py
--------------------------------------------------
ไฟล์นี้รวบรวม "รหัส Error" ทั้งหมดของ SecuGen SDK

หน้าที่:
- แปลผลลัพธ์ที่ได้จากทุกฟังก์ชันของ SDK
- ใช้ตรวจสอบว่าการทำงานสำเร็จหรือผิดพลาด

กลุ่ม Error หลัก:
- Error ทั่วไป (โหลดไลบรารี, พารามิเตอร์ผิด)
- Error ระดับอุปกรณ์ (ไม่พบเครื่อง, timeout, USB)
- Error การสร้าง / เทียบ template

ใช้ตอนไหน:
- ทุกครั้งหลังเรียกฟังก์ชันของ SDK

การประยุกต์ใช้งาน:
- แสดงข้อความ error บนหน้าจอ Kiosk
- เขียน log เพื่อ debug
- แจ้งเจ้าหน้าที่เมื่อเครื่องมีปัญหา


ไฟล์: sgfdxsecuritylevel.py
--------------------------------------------------
ไฟล์นี้กำหนด "ระดับความเข้มงวด" ในการเปรียบเทียบลายนิ้วมือ

หน้าที่:
- ควบคุมว่าการ MATCH จะเข้มงวดแค่ไหน

ระดับตัวอย่าง:
- SL_LOW       = ผ่านง่าย
- SL_NORMAL    = ค่าแนะนำ (สมดุล)
- SL_HIGH      = เข้มงวดมาก

ใช้ตอนไหน:
- ตอนเรียก MatchTemplate()

การประยุกต์ใช้งาน:
- Attendance → LOW / NORMAL
- Login → NORMAL
- งานความปลอดภัยสูง → HIGH


ไฟล์: sgfplibtest.py
--------------------------------------------------
ไฟล์นี้เป็น "ตัวอย่างการใช้งานเต็มรูปแบบ" จาก SecuGen

หน้าที่:
- ทดสอบ SDK แบบครบวงจร
- สแกนลายนิ้ว 2 ครั้ง
- สร้าง Template
- เทียบลายนิ้ว
- แสดง Matching Score
- บันทึกไฟล์ภาพ (.raw) และ template (.min)

ใช้ตอนไหน:
- ทดสอบว่า SDK + เครื่องทำงานปกติ
- ใช้เป็นต้นแบบในการเขียนโปรแกรมจริง

การประยุกต์ใช้งาน:
- แยก logic ไปทำ enroll.py
- แยก logic ไปทำ verify.py
- ใช้ debug เมื่อระบบมีปัญหา


ไฟล์: autoon.py
--------------------------------------------------
ไฟล์นี้เป็นตัวอย่างการใช้งานโหมด Auto-On

หน้าที่:
- สแกนลายนิ้วทันทีเมื่อวางนิ้ว
- ไม่ต้องกดปุ่มหรือ Enter

ใช้ตอนไหน:
- Kiosk
- ระบบอัตโนมัติ
- ไม่มี Keyboard / Mouse

ข้อจำกัด:
- ใช้ได้เฉพาะอุปกรณ์บางรุ่น (เช่น Hamster Plus)

การประยุกต์ใช้งาน:
- ตู้บริการอัตโนมัติ
- เครื่องตรวจสอบตัวตนแบบ Self-service
